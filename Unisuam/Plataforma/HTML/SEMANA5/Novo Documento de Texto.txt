1- Doctype é uma declaração utilizada para que os navegadores saibam em qual versão de html a página utiliza 

2- 
HTML 4.01 Strict: Projetado para garantir a compatibilidade com versões anteriores do HTML.
HTML 4.01 Transitional: Projetado para facilitar a transição de documentos HTML legados para versões mais recentes
HTML 4.01 Frameset: Específico para documentos HTML que usam frames (conjuntos de quadros) para dividir a janela do navegador em várias partes independentes.
XHTML 1.0 Strict: Exige o uso de elementos semânticos corretos e proíbe o uso de elementos obsoletos.
XHTML 1.0 Transitional: Projetado para facilitar a transição de documentos HTML legados para XHTML.
XHTML 1.1: Define um conjunto mais restrito de regras do que o Transitional, focando na semântica e na separação de conteúdo e apresentação.
XHTML Mobile 1.0: Oferece suporte a recursos específicos para ambientes móveis, como telas pequenas e navegadores limitados.
HTML 5: Não é necessária a especificação do doctype. Utiliza novas "tags" e criou o valor "semântico" para as páginas webs.

SEMANA 5

Durante a semana 5 da segunda competência, me ensinaram parte da história do HTML, que foi criado por Tim Berners-Lee. Mais tarde, surgiram as TAGs proprietárias para o navegador Netscape. A padronização do HTML veio com a W3C, que lançou uma versão oficial retirando as tags proprietárias. Posteriormente, o CSS foi consolidado como uma linguagem de estilo para páginas HTML.
Um dos pontos fundamentais que absorvi foi o uso e a importância do doctype em relação à compatibilidade. Ele é crucial para que o navegador interprete a página de forma correta, e entre as versões mais relevantes estão: HTML 4.01 Strict, HTML 4.01 Frameset, XHTML 1.0 Strict, XHTML 1.0 Transitional, XHTML 1.1, XHTML Mobile 1.0 e HTML5.
Como parte do aprendizado prático, fui desafiado a criar uma tabela de cores. Essa atividade proporcionou uma excelente oportunidade para aplicar os conhecimentos em situações semelhantes às reais, consolidando ainda mais o entendimento dos conceitos abordados.

Além disso, comecei a aplicar o CSS para controlar a formatação e o posicionamento dos elementos HTML, percebendo a importância da estilização para a apresentação visual do conteúdo. Na Unidade de Aprendizagem seguinte, aprofundei meus conhecimentos em CSS3, aprendendo a aplicar formatações mais avançadas e desenvolver classes e seletores de ID. Também me apresentaram  o uso de Media Query para criar layouts responsivos, adaptáveis. Ao final meu entendimento de css e html foi consolidado. Em um desafio prático, eu pude aplicar os conhecimentos me passados onde destacou a importância da habilidade e familiaridade com HTML e css.

Além disso, meus conhecimentos sobre formulários foram aprofundados, aprendendo sobre os diversos tipos de entradas de dados que podem ser utilizados em páginas HTML.

Ao final desta unidade de aprendizagem, fiquei apto a conceituar CSS, compreender suas vantagens. Além disso, adquiri habilidades para reconhecer e desenvolver formulários em páginas web, aplicando também técnicas de validação para garantir que os dados inseridos sejam os desejados.
